<task type="auto">
  <name>Task 1: Add GET /api/hooks endpoint to server.js</name>
  <files>ui/server.js</files>
  <action>
Add a `readGlobalHooks()` function near the other settings-related functions (after `stripProjectPeonHooks`). It should:
1. Read `GLOBAL_SETTINGS_PATH` (already defined as `resolve(homedir(), ".claude", "settings.json")`)
2. If file does not exist, return `{ hooks: {} }`
3. Parse JSON. If corrupt, return `{ hooks: {}, error: "Could not parse settings.json" }`
4. Return `{ hooks: settings.hooks || {} }`

Add a route handler in `handleApi()` — insert it before the existing `/api/config` handler:
```js
if (path === "/api/hooks" && req.method === "GET") {
  try {
    const result = readGlobalHooks()
    return Response.json(result)
  } catch (error) {
    return Response.json({ hooks: {}, error: error?.message ?? "Unknown error" })
  }
}
```

Do NOT filter out non-peon hooks — return the raw hooks object. The `_claude_peon` marker on each group is sufficient for the UI to distinguish peon vs non-peon.
  </action>

    <manual>Start the UI server with `bun run ui`, then visit http://localhost:3456/api/hooks in a browser and confirm it returns a JSON object with a `hooks` key containing event names as keys and arrays of hook groups as values.</manual>
  </verify>
  <done>GET /api/hooks returns the full hooks object from ~/.claude/settings.json, including both peon and non-peon hook groups, with no filtering.</done>

<task type="auto">

  <files>ui/index.html</files>
  <action>
Add an "Active Hooks" read-only panel to the UI. This panel should appear ABOVE the mappings section (between the toolbar/controls area and the mappings list) so it is visible on page load.

**HTML structure:** Add a collapsible section with id `active-hooks-section`:
- Header: "Active Hooks" with a subtitle like "Currently registered in ~/.claude/settings.json"
- Body: a container div `#active-hooks-list` that will be populated by JS

**CSS:** Style the panel to match the existing dark theme (use existing CSS variables: `--bg-secondary`, `--bg-tertiary`, `--border-color`, `--text-secondary`, `--accent`, etc.):
- Panel has same card styling as existing sections (border, radius, background)
- Each hook event is a row/card showing the event name and hook groups
- Peon hook groups get a small green accent badge/tag saying "peon" (use `--accent` color)
- Non-peon hook groups get a neutral "external" badge (use `--text-muted` color)
- Show the command string for each hook group (truncated if very long)
- If no hooks are registered, show a muted "No hooks registered" message

**JavaScript:** Add a `loadActiveHooks()` function:
1. Fetch `GET /api/hooks`
2. Parse the response `hooks` object
3. For each event key in the hooks object, render the event name and its groups
4. For each group, check `group._claude_peon === true` to determine badge type
5. Show the `group.hooks[0].command` (the command string) for each group — truncate to ~80 chars with ellipsis if longer
6. Show `async` badge if `group.hooks[0].async === true`

Call `loadActiveHooks()` at page load — add it alongside the existing `loadConfig()` / `loadMeta()` calls in the initialization section (look for `Promise.all` or sequential fetch calls at the bottom of the script).

This panel is READ-ONLY. No edit buttons, no delete buttons, no interaction. It is purely informational.
